# UI 设计方案：个人公开时间线

## 设计方向

极简日志风（选项 B）。气质定义：干净的 feed、时间戳驱动、低信息密度、无衬线字体。像一个有条理的人在安静地记录。

## 设计系统

### 色彩

```css
--bg: #fff;
--text: #1d1d1f;
--text-muted: #86868b;
--border: #f0f0f0;
--accent: #0071e3;
--tag-bg: #f5f5f7;
```

单色系：层级灰 + 一个蓝 accent。绿色系留给 GitHub 热力图，成为首页唯一的色彩焦点。

### 字体

- 正文：`Inter` / `Noto Sans SC` / `-apple-system` / `sans-serif`
- 等宽：`JetBrains Mono`（仅日期、代码、终端暗示元素）

### 间距

`xs: 4px` / `sm: 8px` / `md: 16px` / `lg: 24px` / `xl: 32px` / `2xl: 48px` / `3xl: 64px`

### 容器

- 正文列：`max-width: 600px`，单列居中
- TOC 侧栏出现时（文章详情）：`max-width: 920px`，`grid-template-columns: 160px 1fr`

### 响应式

- `>960px`：文章详情出现 TOC 侧栏
- `≤960px`：TOC 折叠为横向标签，单列
- `≤600px`：满宽 + 16px padding，字号缩小

## 页面设计

### 首页 `/`

从上到下顺序：

1. **Header**：站点标题 + 描述 + 导航（首页/近况/文章/动态）
2. **Now 模块**：`#fafafa` 浅灰背景卡片，三行键值对（做/读/听）
3. **Contributions**：GitHub 热力图。绿色 5 级色阶（`#ebedf0` → `#216e39`），含月份标签和图例。实际数据从 GitHub API 或 scraping 获取
4. **最新近况**：取最近 4-5 条，文章和动态混排。文章显示标题+摘要+标签；动态显示正文+横向滚动缩略图（80×80）
5. **精选项目**：手写列表，项目名+简介
6. **联系**：GitHub / Email / RSS 链接
7. **Footer**：版权信息

### 近况 `/updates`

核心内容页。和首页"最新近况"区结构一致，但全量展示。按日期分组（日期分隔符 `2026-05-02`），文章和动态混排。支持通过标签过滤（React island `TagFilter`）。置顶文章在顶部，最多 5 条。

### 文章 `/articles`

文章型近况的筛选视图，布局同 `/updates` 但只显示 `type:article`。

### 文章详情 `/articles/[slug]`

两栏布局（`>960px`）：左侧 TOC sticky 侧栏 + 右侧正文。

- **Header**：类型标签（ARTICLE）+ 标题 + 描述 + 元信息（日期、阅读时间）+ 标签
- **Cover**：可选封面图，16:9
- **Body**：Markdown 渲染，15px/1.85 行高。h2/h3 带 id 锚点
- **Issue source**：灰底提示条，链接到原始 Issue
- **评论区**：Issue comments 的静态渲染（构建时从 API 拉取）。Avatar + 用户名 + 时间戳 + 正文。底部 GitHub 登录入口

TOC 行为：
- 桌面：sticky 定位，滚动时高亮当前 heading（IntersectionObserver）
- 移动：折叠为横向标签列表，只显示 h2

### 动态 `/moments`

动态型近况的筛选视图。图片优先，每张缩略图 80×80 横向滚动。

### 动态详情 `/moments/[number]`

单列布局（无 TOC）。

- **Header**：类型标签（MOMENT，绿色）+ 时间 + 正文 + 标签
- **Images**：核心内容。单图全宽 4:3；多图 2 列正方形网格
- **Issue source**：同上
- **评论区**：同上

### 标签 `/tags/[tag]`

混合展示带该标签的文章和动态，按时间线排列。

### 404

简洁的 404 页面："页面不存在" + 返回首页链接。需要处理 `/home/` base path。

## 组件树

```
src/
  layouts/
    BaseLayout.astro        # <html>, <head>, SEO, footer
  components/
    SiteHeader.astro        # 站点标题 + 导航
    NowModule.astro         # Now 模块（从 config.ts 读取）
    GitHubHeatmap.astro     # 热力图（静态渲染或 client:load）
    UpdateItem.astro        # 近况条目（根据 kind 分派卡片）
    ArticleCard.astro       # 文章卡片（标题 + 摘要 + 标签）
    MomentCard.astro        # 动态卡片（正文 + 缩略图列表）
    MomentImages.astro      # 动态图片网格（单图/多图）
    ImageGrid.astro         # 通用图片网格
    TagFilter.tsx           # 标签筛选（React island）
    TableOfContents.astro   # TOC 侧栏
    CommentSection.astro    # 评论区（静态渲染）
    CommentItem.astro       # 单条评论
    IssueSource.astro       # "本文由 Issue #X 生成"
    Seo.astro               # SEO meta tags
  pages/
    404.astro
    index.astro
    updates/index.astro
    articles/index.astro
    articles/[slug].astro
    moments/index.astro
    moments/[number].astro
    tags/[tag].astro
```

## 热力图实现策略

第一版用静态 HTML 渲染。构建时通过 GitHub GraphQL API 拉取近 6 个月 contribution 数据（需 `GITHUB_TOKEN`，权限 `read:user`），在 Astro 构建阶段生成热力图 HTML。如果 API 请求失败（如 token 未配置），降级为占位文案 "GitHub 活跃数据暂不可用"，不阻断构建。不引入客户端 API 调用。

色阶：`#ebedf0`（无贡献）→ `#9be9a8`（1-3） → `#40c463`（4-7） → `#30a14e`（8-12） → `#216e39`（13+），同 GitHub 默认色。

## 评论系统实现策略

第一版（当前范围）：
- 构建时通过 `GET /repos/:owner/:repo/issues/:number/comments` 拉取 Issue comments
- 静态渲染为 HTML，无交互
- 底部显示"通过 GitHub 登录后评论"提示，链接到对应 Issue 页面（`html_url`）

第二版（后续迭代）：
- 接入 GitHub OAuth App
- 客户端调 Issues API 发表评论
- 评论后触发 webhook 重新构建

## 图片处理约定

- 动态缩略图（首页/列表）：80×80，`overflow-x: auto` 横向滚动
- 动态详情单图：全宽，`aspect-ratio: 4/3`
- 动态详情多图：2 列网格，`aspect-ratio: 1`，`gap: 4px`
- 文章封面：全宽，`aspect-ratio: 16/9`
- 所有图片在尺寸未知时使用 `aspect-ratio` 占位，避免加载跳动

## 范围边界

第一版实现：
- 以上全部页面和组件
- 首页、近况、文章、文章详情、动态、动态详情、标签、404
- 评论区只读静态渲染
- 热力图静态渲染
- TOC 自动生成 + 滚动高亮
- 基础 SEO（title、description、og、twitter card）

第一版不做：
- 暗色模式切换（CSS 变量已预留，但只实现亮色）
- 无限滚动 / 客户端分页
- 图片灯箱
- 评论交互（登录/发表）
- Pagefind 搜索
- 命令面板
